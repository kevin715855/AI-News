# Digest Quality

## Reader bar
A good digest lets a Vietnamese reader understand the repository in a few minutes without opening the original README.

## Required content
- What the project does
- Why it is notable
- Main technologies or operating shape
- 3-5 concise highlights
- A localized README that remains technically trustworthy

## Quality checks
- Prefer natural Vietnamese over literal translation.
- Preserve code blocks, URLs, commands, package names, API names, and filenames exactly.
- Remove filler; every paragraph should help the reader decide whether the repo matters.
- Reject output that merely wraps the original README with a thin preface.
